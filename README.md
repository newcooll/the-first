## 📖 项目简介

传统的系统清理工具往往充满不可控的“一键清理”黑盒，甚至通过滥用高危指令（如 `/ResetBase`）破坏系统的更新回滚能力。

**CDriveMaster** 采用全新的**规则驱动架构 (Rule-Driven)** 与 **MVVM 状态机防呆设计**。它将应用识别与清理逻辑彻底解耦为透明的 JSON 配置文件，并在每一次物理删除前建立严格的二次确认与预览机制。每一字节的删除，都会生成精确到毫秒的本地审计日志。

## ✨ 核心特性

### 1. 🧠 规则驱动的清理引擎 (Zero-Code Extension)
彻底告别硬编码！通过扫描 `Rules/*.json` 自动生成清理 UI。
- **通配符支持**：原生支持 `*/` 路径展开，无损穿透 WeChat (`wxid_*`)、QQ 等多账号数据隔离目录。
- **高可扩展性**：新增任何应用的清理支持，只需编写数十行 JSON 规则，**无需修改一行 C# 代码**。

### 2. 🛡️ 渐进式安全执行护栏
将垃圾文件严格划分为两大风险等级，杜绝误删：
- 🟢 **SafeAuto (安全缓存)**：浏览器 Cache、日志等可再生文件，二次确认后安全移入回收站。
- 🟡 **SafeWithPreview (需预览内容)**：微信视频、QQ 接收图片等潜在高价值文件。采用 **UI 虚拟化**技术渲染十万级海量文件列表，**用户不物理打勾，绝对不执行删除**。

### 3. ⚙️ 原生的系统级组件维护
安全调用 Windows 底层 DISM (部署映像服务和管理) 接口清理 `WinSxS` 组件库。
- **强制阻断高危参数**：底层硬编码隔离 `/ResetBase` 参数，确保系统在清理后仍可正常卸载 Windows 更新补丁。
- **单向状态机锁**：必须完成分析 -> 明确建议清理 -> 勾选免责声明，才能解锁物理执行按钮。

### 4. 🔬 工业级排障与审计闭环
- **留痕执行**：每一次成功清理都会在 `Logs` 目录下生成工整的 JSON 审计凭证。
- **一键诊断导出**：遇到 Bug？一键打包生成包含环境快照、审计日志、当前规则的 `.zip` 诊断包，让排障变成开卷考试。

## 🚀 下载与运行 (Beta 测试)

本项目采用 **单文件自包含 (Self-Contained Single File)** 模式发布。目标机器**无需**预装 .NET 8 运行时即可直接运行。

1. 前往 [Releases](../../releases) 页面下载最新的 `CDriveMaster_vX.X.X-beta_win-x64.zip`。
2. 解压到任意目录（请确保解压后的文件夹内包含 `Rules` 目录）。
3. ⚠️ **右键点击 `CDriveMaster.exe` -> 以管理员身份运行**。

> **防误杀声明**：由于本项目未挂载企业级 EV 代码签名证书，单文件打包的 .NET 程序大概率会被 Windows Defender 或其他杀毒软件拦截（提示 SmartScreen 保护）。请点击“更多信息” -> “仍要运行”。

## 🛠️ 架构设计

本工程严格遵循 **开闭原则 (OCP)**：
- **UI 层**: WPF + `CommunityToolkit.Mvvm` (完全无 Code-Behind 业务逻辑)。
- **核心层**: 隔离 I/O 操作，所有高危状态通过 `CanExecute` 委托进行依赖属性防呆。
- **测试覆盖**: 核心管线、规则引擎、边界 I/O 均包含 xUnit 单元测试与沙箱集成测试支持。
