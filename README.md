# 🚀 CDriveMaster (C 盘清理大师) v1.0.0

**CDriveMaster** 是一款基于 .NET 8 WPF 构建的高性能、生产级 C 盘空间管理工具。它不只是简单的文件删除脚本，而是一个集成了 **$O(K)$ 内存排序、系统级异步调度、以及物理级防御清理逻辑** 的工程实践方案。

## 🛡️ 技术护城河 (Technical Highlights)

### 1. 极致的内存控制 ($O(K)$ 空间复杂度)
在扫描数百万级文件的大型文件系统时，传统工具往往会因为全量加载文件对象导致 OOM。
- **算法实现**：采用 `PriorityQueue<T, TKey>`（最小堆）维护 Top 20 榜单。
- **数学边界**：空间复杂度锁定在 $O(K)$（$K=20$），无论扫描 $10^5$ 还是 $10^7$ 个文件，内存占用曲线始终保持平稳。
- **时间效率**：插入操作为 $O(\log K)$，整体扫描时间复杂度为 $O(N \log K)$。

### 2. 多重防御性清理协议
针对 Windows `Temp` 目录的复杂性，我们构建了三层安全漏斗：
- **重解析点 (Reparse Point) 阻断**：识别并自动跳过符号链接（Symbolic Link）和挂载点，防止清理逻辑误入非目标驱动器。
- **后序遍历 (Bottom-Up) 删除**：先清空子目录文件，再销毁空文件夹，确保删除动作的原子性与可追溯性。
- **1 小时修改保护期**：自动识别正在被系统或应用写入的临时文件，强制执行 3600 秒的冷却保护。

### 3. 原生系统交互 (Shell Interop)
- **Shell API 握手**：通过 P/Invoke 调用 `shell32.dll` 接口（`SHQueryRecycleBin`, `SHEmptyRecycleBin`），实现与 Windows 原生回收站的高效通信。
- **受控删除**：手动删除功能通过 `Microsoft.VisualBasic` 接口封装，支持“移至回收站”逻辑，为用户提供“后悔药”机制，而非暴力抹除物理扇区。

## 🚀 快速上手与部署

本项目采用 **自包含 (Self-contained) & 单文件 (Single-file)** 模式发布，无需安装 .NET 运行时。

### 开发环境构建
```powershell
dotnet build -c Debug
```

### 生产级发布流水线
我们内置了具备 **6 秒存活冒烟测试** 的发布脚本：
```powershell
.\build-release.ps1 -OpenOutput
```
该脚本会自动处理发布产物目录清理、资产还原、单文件编译、冒烟测试与 ZIP 归档。

## 👨‍💻 开发过程总结：从 0 到 1.0.0 的工程进阶

回顾整个开发周期，我们不仅是在写功能，更是在对抗 **Windows 环境的不确定性**。

### 1. 核心开发阶段 (The Core)
- **经验点**：最初我们面对的是如何从千万级文件中筛选出最大的 20 个。通过引入 `PriorityQueue`，我们将内存占用从“跟随文件量增长”压制到了“常量级锁定”。
- **感悟**：**算法是稳定性的第一道防线。**

### 2. UI 与异步调度阶段 (The UI)
- **经验点**：WPF 的 UI 线程极其脆弱。我们通过 `Task.Run` 隔离扫描逻辑，并使用 `IProgress<T>` 实现高频数据回传。
- **感悟**：**流畅度不是性能问题，是调度问题。** 只要 UI 不死锁，用户对耗时的容忍度会大幅提升。

### 3. 系统兼容性与防御阶段 (The Defense)
- **经验点**：在处理 `C:\Windows\Temp` 时，遇到了权限拒绝和符号链接死循环。我们通过配置 `EnumerationOptions` 实现了静默跳过，并建立了“后序遍历删除”逻辑。
- **感悟**：**系统级工具的平庸与卓越，区别在于对 Edge Cases 的处理。**

### 4. 发布与环境抗性阶段 (The Delivery)
- **经验点**：在这个阶段我们遭遇了最顽固的敌人：**文件锁**和**网络波动**。通过开发/发布解耦策略与 `build-release.ps1` 自动化脚本，我们抹平了不同机器上的环境差异。
- **感悟**：**没经过自动化流水线检验的代码，不叫“已完成”。**
