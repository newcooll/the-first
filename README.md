# CDriveMaster v1.0.0

一款基于 $O(K)$ 内存复杂度的生产级 C 盘清理工具。

## 技术护城河

- $O(K)$ 空间复杂度排序：使用最小堆 PriorityQueue 维护 Top 20 文件，避免全量加载文件集合。对海量文件场景，内存占用上界固定为 $K$，整体复杂度为 $O(N \log K)$。
- 自愈式单文件发布架构：启用 IncludeNativeLibrariesForSelfExtract，强化单文件在复杂 Windows 环境下的启动稳定性，降低 c000041d 相关风险。
- 防御性删除逻辑：采用后序遍历 (Bottom-Up) 清理策略，配合 1 小时修改时间保护阈值，避免误删正在使用的临时文件。
- 黑匣子崩溃审计：应用在启动阶段挂接全局异常捕获，将异常写入 %AppData%/CDriveMaster/logs，支持用户侧问题追溯。

## 快速交付

```powershell
.\build-release.ps1 -OpenOutput
```

发布脚本默认执行以下流程：

- 运行时资产还原 (win-x64)
- 单文件自包含发布
- 6 秒存活冒烟测试
- 产物打包为 ZIP 归档

## 工程心得

1. 算法不仅是性能手段，也是稳定性屏障。以最小堆实现 $O(N \log K)$，避免全量排序的高内存压力。
2. 交付不仅是代码通过编译，还包括对权限、文件锁、发布模式差异的环境抗性设计。
3. 用自动化脚本替代重复手工操作，让诊断、发布、回归形成可复用流水线。
