# CDriveMaster 用户帮助手册 (Beta 版)

欢迎使用 CDriveMaster。本工具致力于提供透明、安全、规则驱动的 C 盘冗余清理方案。与市面上传统的“黑盒”清理工具不同，我们拒绝静默扫描和流氓删除，确保您的每一字节数据都删得明明白白。

## 1. 核心清理模式说明

我们在底层将所有可清理的冗余文件划分为两个严格的风险等级：

### 🟢 SafeAuto (安全自动清理)

定义：绝对安全的程序运行缓存（如 Chrome 的 GPUCache、网页缓存等）。

行为：点击“一键清理”后，系统会弹出二次确认框，告知您即将释放的总大小。确认后直接移入回收站。

### 🟡 SafeWithPreview (需预览的安全清理)

定义：可能包含用户个人价值的数据（如 WeChat 的视频缓存、QQ 的接收图片等）。

行为：点击清理会弹出一个详细的复选列表。您可以看到每一个文件的绝对路径、大小和最后访问时间。只有您手动打勾的文件，才会被真正清理。

## 2. 常见使用场景与解惑

### 为什么点击“扫描”后界面没反应？

如果您点击扫描后，界面瞬间恢复原状且没有弹出任何提示，通常是因为：

- 目标应用未安装在系统盘：例如您的微信/QQ 安装或数据存储重定向到了 D 盘，而当前规则仅扫描默认路径。
- 缓存极小或已被清理：底层扫描到的可清理体积为 0 Byte。

(注：我们将在后续版本中优化此处的弹窗提示，避免造成“程序卡死”的错觉。)

### 为什么“系统组件分析/清理”速度非常慢？

系统维护模块调用的是 Windows 原生的底层 DISM (部署映像服务和管理) 接口。

- 分析阶段：系统需要比对成千上万个 Windows 更新 (WinSxS) 的版本差异，物理耗时通常在 3 到 15 分钟不等。
- 清理阶段：底层正在安全地卸载被取代的旧组件包。

请保持耐心，在此期间程序并未卡死。切勿强制关闭窗口，以免中断系统底层服务。

### 为什么你们的系统清理不提供“深度极限瘦身”？

市面上许多工具通过强制删除所有备份或使用 /ResetBase 参数来榨干 C 盘空间。这极其危险！

CDriveMaster 永远不会使用破坏性参数。我们的系统清理严格保障了您未来遇到“Windows 蓝屏或更新引发 Bug”时，依然可以通过控制面板卸载最近的更新来回滚系统。安全永远高于多释放出的那几个 G 空间。

### 为什么找不到以前的“大文件”和“Temp 垃圾”清理了？

CDriveMaster 刚刚完成底层的“泛化规则引擎”重构（V2 架构）。为了保证架构的极度稳定，我们暂时下线了全局大文件搜索和部分系统临时文件的扫描。这些功能将在后续版本（v0.7+）中以更安全、更直观的方式回归。

## 3. 审计、排障与错误反馈

CDriveMaster 秉承“死后可验尸”的工程底线。

### 📄 审计日志 (Audit Logs)

每一次您点击确认执行的物理清理动作，无论成功还是失败，都会在本地生成一份包含时间、大小、退出码的 JSON 日志。您可以点击软件底部的“📄 审计日志”随时查阅。

### 📦 一键导出诊断包 (提交 Bug 必读)

如果您遇到了扫描报错、清理失败或其他异常：

1. 点击软件底部的 “📦 导出诊断包”。
2. 程序会在您的桌面生成一个名为 CDriveMaster_Diag_日期_时间.zip 的文件。
3. 该压缩包内仅包含您的 Windows 基础版本信息、本次运行日志以及清理规则副本（绝不包含您的任何私人文件内容）。

请将此 .zip 文件附带在您的反馈或 GitHub Issue 中，这将帮助开发者在一分钟内定位您的环境问题。
