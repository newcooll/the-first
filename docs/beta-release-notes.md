# CDriveMaster v0.6.0-beta 发布说明

## 版本概览
CDriveMaster v0.6.0-beta 是首个基于规则驱动架构的 Beta 版本。本版本在保持清理能力可扩展性的同时，重点强化了执行安全性、可追溯性与发布可交付性，为后续正式版打下稳定基础。

## 核心特性

### 1. 规则驱动引擎
- 通过规则文件驱动清理策略与目标路径解析，降低硬编码耦合。
- 当前已支持 WeChat、QQ、Chrome 缓存的无损识别与清理流程。
- 规则加载具备容错能力，单条规则异常不会导致应用整体不可用。

### 2. 安全系统瘦身
- 封装并执行 Windows DISM StartComponentCleanup 流程，用于系统组件存储清理。
- 严格拒绝危险参数 ResetBase，避免破坏更新回滚能力。
- 提供分析、确认、执行的受控闭环，降低误操作风险。

### 3. 执行审计追踪
- 所有物理删除动作与系统维护执行结果均生成本地 JSON 审计日志。
- 审计日志可用于问题追踪、回归复盘与安全合规留痕。
- 支持通过界面快捷入口直接打开日志目录。

## 已知限制 (Known Limitations)
- 需以管理员身份运行。否则系统清理模块会被权限策略拦截。
- 程序体积较大（约 150MB+）。这是采用 .NET 8 自包含单文件部署的预期代价。
- 当前安装包未进行数字签名，可能触发 Windows Defender 等安全软件拦截，需要手动放行。

## 用户反馈指南
为提升问题定位效率，请在提交 Issue 时尽量附带以下信息：
- Logs 目录下对应时间段的执行审计日志（JSON 文件）。
- 操作系统版本信息（例如 Windows 10 22H2 / Windows 11 23H2）。
- 问题发生时的界面截图与关键报错描述。
- 复现步骤与是否以管理员权限运行。
